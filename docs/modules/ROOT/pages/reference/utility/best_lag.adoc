= Best Lag

include::../../common.adoc[]

== Overview

`best_lag` finds the lag at which the recent signal most resembles itself. For each candidate lag in a range, it scores the newest `window` samples against the samples `lag` back by *normalized cross-correlation* (NCC): their dot product divided by their energies, so the score ignores level. A score of 1 is a perfect shape match, 0 is no correlation.

On a periodic signal the score peaks when the lag equals the period: the newest window and the window one period back are the same shape.

image::best_lag_selfsim.svg[alt="Self-similarity at lag = period", width=700px, align=center, title="On a periodic signal the NCC peaks at lag = period: the newest window matches the window one lag back", link={imagesdir}/best_lag_selfsim.svg]

The best integer lag is then refined to *sub-sample precision* by a parabola fit through it and its two neighbors:

image::best_lag_ncc.svg[alt="Normalized cross-correlation vs lag", width=700px, align=center, title="Zoomed view of the NCC peak (periodic signal, period 100.25): the parabola fit refines the integer best at lag 100 to a sub-sample peak at 100.25", link={imagesdir}/best_lag_ncc.svg]

`best_lag` knows nothing about *pitch*: no fundamental, no harmonics, no octave logic. On a single note the lag is the period; on polyphonic material it is the best common quasi-period in range; on noise the similarity stays low. That makes it a general tool for loop and splice-point selection (sustain and freeze effects, WSOLA) and grain re-anchoring (PSOLA), with the similarity score doubling as a confidence measure of how loopable the material is at that lag.

NOTE: Cost is O(window × range): `best_lag` is intended for occasional, event-driven use (per hold, per splice, per grain), not per sample. In an audio callback, prefer a buffer whose integer indexing is cheap (analysis does not need interpolated reads).

== Include

```c++
#include <q/utility/best_lag.hpp>
```

== Declaration

```c++
struct lag_result
{
   float    lag;
   float    similarity;
};

template <typename Buffer>
lag_result best_lag(
   Buffer const& buf
 , std::size_t window
 , std::size_t min_lag
 , std::size_t max_lag
);
```

== Expressions

=== Notation

`buf`         :: A buffer indexable with `std::size_t`, where index 0 refers to the newest element, e.g. `{ring_buffer}`.
`window`      :: Comparison window length, in samples.
`min_lag`, `max_lag` :: The inclusive lag search range, in samples.
`r`           :: A `lag_result`.

=== Function Call

[cols="1,1,1"]
|===
| Expression                                | Semantics                                | Return Type

| `best_lag(buf, window, min_lag, max_lag)` | Search [`min_lag`, `max_lag`] for the
                                              lag at which the newest `window`
                                              samples best match themselves
                                              `lag` samples ago.                      | `lag_result`
| `r.lag`                                   | The best lag, sub-sample refined.        | `float`
| `r.similarity`                            | Normalized correlation at the peak:
                                              1 = perfect match, 0 = uncorrelated.    | `float`
|===

WARNING: The buffer must hold at least `window + max_lag` samples of history (plus the interpolation margin, if the buffer interpolates fractional indices). Searching beyond the available history is undefined behavior.

== Example

```c++
// On engage: measure the period of the recent signal
auto r = q::best_lag(buff, 2048, sps/500, sps/40);
if (r.similarity > 0.95f)      // loopable?
   start_hold(r.lag);
```
