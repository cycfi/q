== Overview

Three waveshapers for bounding (and coloring) a signal:

* `hard_clip` hard-limits to `Â±max`: values inside the range pass through bit-exactly; values outside are clamped. Zero coloration below the ceiling, hard corners at it (strong high harmonics).
* `cubic_clip` bounds to Â±1 with a smooth cubic (`1.5s âˆ’ 0.5sÂ³` after a Â±1 clamp), so the transfer curve flattens gradually into the rails instead of cornering. Gentle, tape-like saturation with mild odd-harmonic coloration near the rails. (The cubic is a polynomial fit of a quarter sine -- very close to `sin(Â½Ï€Â·s)`, but cheaper and `constexpr`.)
* `tanh_clip` bounds to Â±1 with a tanh (the exp-based `fast_tanh`). tanh saturates over all reals, so there is no input clamp: louder inputs roll off gradually and stay distinguishable instead of pinning flat. It is the cheapest soft clip on a desktop FPU (see <<cost>>); approximate, and not `constexpr`.

The former names `clip` and `soft_clip` remain as deprecated aliases for `hard_clip` and `cubic_clip`.

== Declaration

```c++
struct hard_clip
{
   constexpr      hard_clip(decibel max);
   constexpr      hard_clip(float max = 1.0f);

   constexpr float operator()(float s) const;
};

struct cubic_clip : hard_clip
{
   constexpr float operator()(float s) const;
};

struct tanh_clip
{
   float operator()(float s) const;
};

// Deprecated former names:
using clip [[deprecated]] = hard_clip;
using soft_clip [[deprecated]] = cubic_clip;
```

== Expressions

=== Notation

`c`         :: Object of type `hard_clip`, `cubic_clip`, or `tanh_clip`.
`max`       :: The clamp level: a `float`, or a `decibel` level.
`s`         :: Input sample. A `float`.
