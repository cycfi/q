= Hilbert Quadrature

include::../../common.adoc[]

== Overview

`hilbert_quadrature` takes one real input sample and returns two outputs that hold a `90` degree phase difference across the audio band, the in-phase and quadrature pair that together form the analytic signal. Splitting a signal this way is the starting point for single-sideband modulation, frequency shifting, and reading a signal's instantaneous amplitude and phase: with the two outputs `i` and `q` in hand, `sqrt(i*i + q*q)` is the envelope and `atan2(q, i)` is the instantaneous phase.

image::hilbert_quadrature.svg[alt="Phase difference of the two outputs, and their quarter-cycle time offset", width=700px, align=center, title="Left: the phase difference between the two outputs holds within about a degree of 90 across the entire band, an equiripple that never leaves the useful range. Right: a single tone fed through the filter emerges as two outputs a quarter cycle apart, one riding as cosine while the other rides as sine.", link={imagesdir}/hilbert_quadrature.svg]

Rather than the long FIR a windowed Hilbert transform would need, `hilbert_quadrature` uses two short chains of `{polyphase_allpass}` sections, a design by Olli Niemitalo (see http://yehar.com/blog/?p=368[yehar.com]) that is close to the most efficient structure for the job. The two chains have all-pass magnitude, so both outputs pass every frequency at unity gain; they are built so their phases run a quarter cycle apart. One chain carries an extra one-sample `{delay}` to align the group delay of the pair. The section coefficients are fixed for this band split, so the filter is default constructed with no parameters.

The phase difference is symmetric around Nyquist/2 and stays within a fraction of a degree of `90` across essentially the whole band, degrading only in the last sliver next to DC and Nyquist. Feed it a real signal and read both outputs every sample.

== Include

```c++
#include <q/fx/hilbert_quadrature.hpp>
```

== Declaration

```c++
struct hilbert_quadrature
{
   std::pair<float, float>    operator()(float s);
};
```

== Expressions

In addition to the expressions below, `hilbert_quadrature` is default constructible and copy constructible.

=== Notation

`hq`          :: Object of type `hilbert_quadrature`.
`s`           :: Input sample.

=== Function Call

[cols="1,1,1"]
|===
| Expression   | Semantics                                          | Return Type

| `hq(s)`      | Process `s` and return the `{i, q}` pair: the
                 in-phase output first, the quadrature output
                 second, a `90` degree phase difference apart.      | `std::pair<float, float>`
|===

== Example

```c++
q::hilbert_quadrature  hq;

// ... per sample:
auto [i, q] = hq(s);

// The analytic signal gives the instantaneous envelope and phase
float envelope = std::sqrt(i * i + q * q);
float phase    = std::atan2(q, i);
```
