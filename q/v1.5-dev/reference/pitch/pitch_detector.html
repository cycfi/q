BACF Pitch Detector

Overview

pitch_detector is the top of the pitch-detection stack and the class most applications use directly. It wraps a BACF Period Detector and turns its raw period estimates into a stable frequency in Hz.

A period detector reports a fresh estimate every analysis window, and those estimates can wobble on the attack, jump octaves on harmonically rich notes, or drift as a note decays. pitch_detector adds the logic that makes the output usable as pitch:

  • Frequency: converts the detected period to Hz against the sample rate.

  • Bias and hysteresis: prefers the established frequency unless the new estimate is convincingly different, so a steady note stays put instead of flickering between a fundamental and its harmonic.

  • Smoothing: a short median tames single-frame outliers.

  • Note-shift detection: recognizes when a genuinely new note has started (is_note_shift()), so attacks are not mistaken for pitch glitches.

  • Prediction: predict_frequency() offers a low-latency estimate ahead of the confirmed result, useful when latency matters more than certainty.

Figure 1. As a note rings, the 2nd harmonic’s periodicity can overtake the fundamental’s (top). Without history the reported frequency jumps up an octave at the crossover; the bias holds it on the true fundamental (bottom)

The detection range and sample rate are fixed at construction; everything else is reported per sample.

Include

#include <q/pitch/pitch_detector.hpp>

Declaration

class pitch_detector
{
public:

   static constexpr float  max_deviation = 0.90f;
   static constexpr float  min_periodicity = 0.8f;

                           pitch_detector(
                              frequency lowest_freq
                            , frequency highest_freq
                            , float sps
                            , decibel hysteresis
                           );

   bool                    operator()(float s);
   float                   get_frequency() const;
   float                   predict_frequency(bool init = false);
   bool                    is_note_shift() const;
   std::size_t             frames_after_shift() const;
   float                   periodicity() const;
   void                    reset();

   bitset<> const&         bits() const;
   zero_crossing_collector const& edges() const;
   bacf_period_detector const&    get_period_detector() const;
};

Zero Crossing Collector
get_frequency() returns 0 until the first confident lock. Use periodicity() (compare against min_periodicity) to gate consumers, and is_note_shift() to react to new notes.

Example

q::pitch_detector pd{q::frequency{50}, q::frequency{500}, sps, -30_dB};

for (auto s : samples)
{
   if (pd(s))                                   // frequency updated
   {
      if (pd.is_note_shift())
         on_new_note();
      if (pd.periodicity() > q::pitch_detector::min_periodicity)
         float hz = pd.get_frequency();
   }
}
