Zero Crossing

Overview

zero_crossing reports whether a signal is currently above or below zero, emitting a pulse that stays high for the positive part of each cycle. It is the front-end that turns a continuous waveform into the clean, rectangular edges that period and timing analysis work from. To keep noise near the zero line from producing spurious transitions, it applies hysteresis: the output goes high when the signal rises above zero, but only releases once the signal falls back below a negative threshold set by the hysteresis amount.

Figure 1. Left: the boolean pulse (green) goes high while the input is above zero and releases only after it clears the lower edge of the hysteresis band (amber). Right: zero_crossing_ex marks the leading edge of each pulse (magenta) and records the peak height of the waveform it bounds (green), the raw material for period and pulse-width measurement.

The hysteresis may be given as a plain float or as a decibel value; internally the signal is offset by half the hysteresis so the detection stays centered on the true zero. The main operator()(float s) returns the current boolean state, and a nullary operator()() reads that state back without advancing.

zero_crossing_ex is the extended form, built for the timing measurements that pitch detection needs. Its call operator returns an int event code rather than a bool: 1 at the leading edge (the sample where the pulse opens), -1 at the trailing edge (where it closes), and 0 when nothing changed. Each pulse carries an info record, retrieved with get_info(), holding the crossing samples, the peak height of the bounded waveform, and the leading and trailing edge positions. From two consecutive info records it computes the period in whole samples and a fractional_period accurate to a fraction of a sample by linearly interpolating each crossing. The leading edge and crossing data are valid as soon as the pulse opens, which is enough for period; the width and height become valid only after the trailing edge.

The extended info is the same machinery the Zero Crossing Collector uses to propose candidate periods to the pitch detector.

Include

#include <q/fx/zero_crossing.hpp>

Declaration

struct zero_crossing
{
               zero_crossing(float hysteresis);
               zero_crossing(decibel hysteresis);

   bool        operator()(float s);
   bool        operator()() const;
};

class zero_crossing_ex
{
public:

   class info
   {
   public:
      using crossing_data = std::pair<float, float>;

      // Available after the leading edge
      crossing_data     crossing() const;
      std::size_t       leading_edge() const;
      std::size_t       period(info const& next) const;
      float             fractional_period(info const& next) const;
      void              update(float height);

      // Available after the trailing edge
      std::size_t       width() const;
      float             height() const;
      std::size_t       trailing_edge() const;
      bool              similar(info const& next) const;
   };

               zero_crossing_ex(float hysteresis);
               zero_crossing_ex(decibel hysteresis);

   int         operator()(float s);
   bool        operator()() const;
   info const& get_info() const;
};

Example

using namespace q::literals;

// A simple gate that pulses on the positive half of each cycle
q::zero_crossing  zc{-30_dB};

// ... per sample:
bool  positive = zc(s);

// Extract timing information for period measurement
q::zero_crossing_ex  zx{-30_dB};

// ... per sample:
int  edge = zx(s);
if (edge == 1)
{
   // a new pulse opened: its info is now available
   auto const& i = zx.get_info();
   // pair with the previous pulse's info to get a period
}
