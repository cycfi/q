Delay

Overview

example/delay/delay.cpp is the library’s original "Hello, World" (it opens the Quick Start): it loads this example’s own recording of a sustained low E and plays it through a feedback delay: the raw audio in the left channel, the delayed signal in the right, so you can hear exactly what the effect adds.

Figure 1. Feedback delay signal chain: input splits to dry left; output (_y) feeds back through ×0.85 and 350 ms of delay to the summer

The input branches immediately: the dry signal goes straight to the left channel, while the main path feeds a summer (+). The summer mixes the input with whatever the delay line is holding and emits _y to the right channel. _y is also attenuated by 0.85 and pushed back into the delay, so the output of each echo becomes the input of the next.

How It Works

The entire effect is three lines around a Delay:

q::delay _delay{350_ms, sps};    // a fractional delay line

// In the processing loop, per sample s:
_y = s + _delay();               // mix the dry and the delayed signal
_delay.push(_y * _feedback);     // feed the result back in

Because the output (not the input) is pushed back into the delay (attenuated by the feedback factor), every echo spawns the next one: a single delay line produces an infinite, exponentially decaying echo train. The feedback must stay below 1.0; at 0.85 the tail is long and lush, as the figure shows.

The surrounding scaffolding is the standard Q file-playback pattern:

  • q::wav_reader loads the wav into a std::vector<float>; the baked-in AUDIO_DIR points at the example’s own audio/ directory, so this works from any working directory.

  • The processor subclasses Audio Stream with no inputs and two outputs; playback streams generate their input by walking the loaded buffer one sample per frame.

  • main starts the stream, sleeps for the file’s duration (q::duration(wav.length()) / wav.sps()), and stops.

Running

cmake -B build
cmake --build build --target example_delay
build/example/delay/example_delay

Plays the processed file once (dry left, delayed right) and exits. Listen in stereo: the point of the channel split is the comparison.
